## 🔴 **프롬프트 A: M6 판단 문구 보정 (E2E 기준)**

### **역할**
당신은 ZeroSite Decision OS의 LH 실무 검증 AI입니다.

### **목표**
E2E 테스트 결과를 기준으로 M6 Executive Dashboard의 판단 문구를 수정하십시오.

### **수정 규칙**

#### **1. GO 판단 문구**
```
❌ 잘못된 표현:
- "문제 없음"
- "완벽함"
- "승인 가능"

✅ 올바른 표현:
- "관리 가능한 리스크"
- "LH 기준 충족"
- "보수적 설계로 통과 가능성 높음"
```

**이유**: GO도 리스크가 있다. 다만 관리 가능한 수준.

#### **2. CONDITIONAL 판단 문구**
```
필수 포함 문구:
"보완 조건 선행 시 사업 추진 가능"

❌ 잘못된 표현:
- "통과 가능"
- "문제 없음"

✅ 올바른 표현:
- "주차 대수 +5대 확보 시 가능"
- "M5 매입가 상단 3% 이내 권고"
- "M4 주차대수 법정 대비 +5% 확보 필요"
```

**이유**: CONDITIONAL은 "조건 충족 없으면 NO-GO"와 같다.

#### **3. NO-GO 판단 문구**
```
❌ 잘못된 표현:
- "불가능"
- "실패"
- "추천하지 않음"

✅ 올바른 표현:
- "용적률 한도 초과로 건축 불가"
- "LH 매입가 범위 벗어남 (감정가 대비 +15%)"
- "과다 규제로 사업성 미확보"
```

**이유**: 감정적 표현 금지. 구조적 한계 중심 설명.

### **출력 형식**

```markdown
## M6 판단 결과

### 최종 판단
🟢 GO (조건부)

### 1줄 요약
보수적 설계로 LH 심사 통과 가능성 높으나, 주차 대수 +5대 확보 권고

### 판단 근거

#### 1. 구조적 적합성 (M1~M4)
- M1: 토지 면적 충분, 용도지역 2종일반주거 적합
- M3: 도시형생활주택 1순위 선정 (92점)
- M4: 용적률 180% (한도 200%, 안전 여유 10%)

#### 2. 규제 준수 (M2, M4)
- M2: LH 매입가 범위 이내 (감정가 대비 -3.2%)
- M4: 주차 대수 법정 145대 → 현재 140대 (부족 5대)

#### 3. 사업성 확보 (M5)
- 안전 마진 12.76% (목표 10% 이상)
- NPV 양수, IRR 7.5%
- 공실률 5% 가정 시에도 수익성 확보

### 보완 조건
1. 🟡 M4: 주차 대수 +5대 확보 (기계식 주차 검토)
2. 🟡 M5: 매입가 감정가 상단 3% 이내 권고
3. 🟡 M7: 커뮤니티 운영 규칙 사전 명문화
```

### **완료 기준**
- [ ] GO/CONDITIONAL/NO-GO 라벨 명확
- [ ] 1줄 요약 30자 이내
- [ ] 판단 근거 3개 (구조/규제/사업성)
- [ ] 보완 조건 구체적 (CONDITIONAL 시)
- [ ] 감정적 표현 없음

## 🔴 **프롬프트 B: 신뢰도 수치 표현 정교화**

### **목표**
E2E 테스트 결과를 기준으로 신뢰도(%) 표현 방식을 수정하십시오.

### **신뢰도 구간 정의**

```
┌──────────────┬────────────────────────────────────┐
│ 신뢰도 구간  │ 의미                                │
├──────────────┼────────────────────────────────────┤
│ 90% 이상     │ 구조적 제약 거의 없음               │
│              │ LH 심사 통과 가능성 매우 높음        │
├──────────────┼────────────────────────────────────┤
│ 80~89%       │ 관리 가능한 리스크 존재             │
│              │ 보완 조건 충족 시 통과 가능          │
├──────────────┼────────────────────────────────────┤
│ 70~79%       │ 조건부 가능                         │
│              │ 여러 보완 조건 충족 필수             │
├──────────────┼────────────────────────────────────┤
│ 70% 미만     │ 구조적 부적합                       │
│              │ 사업 추진 어려움                     │
└──────────────┴────────────────────────────────────┘
```

### **표현 규칙**

#### **1. 라벨 명시**
```html
<!-- 잘못된 예 -->
<div class="confidence">85%</div>

<!-- 올바른 예 -->
<div class="confidence-container">
  <span class="confidence-label">판단 신뢰도</span>
  <span class="confidence-value">85%</span>
  <span class="confidence-meaning">관리 가능한 리스크</span>
</div>
```

#### **2. 색상 코딩**
```css
/* 90% 이상: 초록색 */
.confidence-high { color: #22C55E; }

/* 80~89%: 노란색 */
.confidence-medium { color: #FBBF24; }

/* 70~79%: 주황색 */
.confidence-low { color: #F97316; }

/* 70% 미만: 빨간색 */
.confidence-critical { color: #EF4444; }
```

#### **3. Tooltip 설명**
```javascript
const confidenceTooltips = {

  low: "조건부 가능. 여러 보완 조건 충족 필수.",

```

## 🟢 **프롬프트 D: 데모 케이스 설명 보강**

### **역할**
당신은 ZeroSite Decision OS의 Demo Experience Designer입니다.

### **목표**
Demo 프로젝트(GO 케이스)에 명확한 설명을 추가하여 사용자가 "왜 GO인지"를 즉시 이해하도록 하십시오.

### **Demo 케이스 기본 정보**
```

최종 판단: 🟢 GO (조건부)
판단 신뢰도: 85%
주요 리스크: 주차 대수 5대 부족
```

### **필수 포함 내용**

#### **1. 왜 GO인지**
```markdown
### 이 프로젝트가 GO인 이유

1. **토지·입지 조건 우수**
   - 2종일반주거지역, 면적 충분 (300m²)
   - 도로 접도 양호 (6m)
   - 주변 인프라 완비

2. **LH 기준 충족**
   - 매입가: 감정가 대비 -3.2% (LH 범위 이내)
   - 용적률: 180% (한도 200%, 안전 여유 10%)
   - 건폐율: 55% (한도 60%)

3. **사업성 확보**
   - 안전 마진 12.76% (목표 10% 이상)
   - NPV 양수, IRR 7.5%
   - 공실률 5% 가정 시에도 수익성 확보
```

#### **2. 어떤 리스크가 있었고 어떻게 관리 가능한지**
```markdown
### 리스크 및 관리 방안

#### 🟡 주차 대수 부족 (5대)
- **현황**: 법정 145대 → 현재 140대
- **관리 방안**: 기계식 주차 5대 추가
- **비용**: 약 5,000만원 (총 사업비의 0.5%)
- **결론**: 관리 가능한 리스크

#### 🟡 매입가 협상 필요
- **권고**: 감정가 상단 3% 이내
- **이유**: 안전 마진 확보
- **협상 전략**: M2 보고서 근거 제시

#### 🟢 커뮤니티 리스크 낮음
- M7에서 민원 유형 예측 완료
- 대응 전략 수립됨
- 입주자대표회의 운영 규칙 사전 명문화 권고
```

#### **3. 이 상태로 LH에 제출해도 되는 이유**
```markdown

1. **LH 기준 충족**
   - 매입가, 용적률, 건폐율 모두 기준 이내
   - 공급유형 적합성 검증 완료
   - 사업성 확보 (안전 마진 10% 이상)

3. **판단 근거 투명**
   - M1~M7 모든 모듈 분석 완료
   - 각 모듈별 근거 명확
   - PDF 보고서로 즉시 제출 가능

⚠️ **제출 전 확인 사항**

- 현장 실사로 데이터 재확인
```
