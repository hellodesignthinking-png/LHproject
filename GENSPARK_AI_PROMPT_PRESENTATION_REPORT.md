# Genspark AI용 수정·보완 프롬프트
## F. 설명용 프레젠테이션 보고서 (Explanation / Presentation Deck)

---

## 🎯 목표

현재 업로드된 **F. 설명용 프레젠테이션 보고서**를 **의사결정 시나리오 중심의 설명용 프레젠테이션**으로 전환하여, 
"이미 나온 분석 결과를 사람이 이해하고 결정하게 만드는 설명 전용 문서"로 재구성한다.

---

## ❌ 현재 템플릿의 구조적 문제 (1:1 대조 결과)

### 1. 정체성 이탈: 의사결정 도구 → 숫자 나열 슬라이드쇼
- **문제:** 설명용 프레젠테이션이 "보고서 요약"처럼 출력되고 있음
- **원인:** 숫자 중심 구조 (토지가치 ○○억, IRR ○○%, NPV ○○억)
- **영향:** "이야기 구조"가 아닌 "데이터 나열"로 전락

### 2. "결론 먼저" 원칙 위반
| 항목 | 현재 템플릿 | 기획 의도 |
|------|------------|-----------|
| Opening | ❌ 제목만 표시 (Slide 1) | ✅ 한 줄 정의 + 권장 판단 + "무엇을 결정해야 하는가" |
| 결론 위치 | ❌ Slide 2: 종합 점수/리스크 수준만 표시 | ✅ Slide 1: 진행/조건부/중단 명확히 표시 |
| 판단 명확성 | ❌ "{{ go_decision }}" (불명확) | ✅ "🟢 진행 권장 / 🟡 조건부 / 🔴 중단" |

### 3. 핵심 슬라이드 누락
| 필수 슬라이드 | 현재 상태 | 기획 의도 |
|--------------|----------|-----------|
| **왜 이 땅인가** | ❌ 없음 | ✅ 사전 검토 통과 이유, 다른 후보지 대비 특징 |
| **사업 구조** | ❌ 없음 | ✅ 돈의 흐름 다이어그램, Exit 구조 (LH) |
| **리스크 요약** | ❌ "리스크 수준"만 표시 | ✅ 주요 리스크 3~4개 + 관리 가능성 |

### 4. 표현 원칙 위반
| 원칙 | 현재 템플릿 | 기획 의도 |
|------|------------|-----------|
| **한 슬라이드 = 한 메시지** | ❌ Slide 4: 공급유형 + 세대수 + 용적률 혼재 | ✅ 한 슬라이드는 하나의 핵심 메시지만 전달 |
| **말로 설명 가능** | ❌ 숫자 나열 (IRR, NPV, 총 투자비) | ✅ "왜?"에 대한 답 중심 |
| **"왜?"에 대한 답** | ❌ 없음 | ✅ 모든 슬라이드는 "왜?"를 설명 |

### 5. 슬라이드 순서 및 구조 문제
현재 템플릿 (6장):
1. ❌ Slide 1: 제목 (정보 부족)
2. ❌ Slide 2: 종합 점수/리스크 수준 (숫자만 표시)
3. ❌ Slide 3: 토지가치 (M2) - 숫자 나열
4. ❌ Slide 4: 개발 규모 (M3+M4) - 숫자 나열
5. ❌ Slide 5: 사업성 (M5) - IRR/NPV 중심
6. ⚠️ Slide 6: 다음 단계 (불충분)

기획 의도 (10~15장):
1. ✅ Opening / 결론 먼저
2. ✅ 대상지 한 장 요약
3. ✅ 왜 이 땅을 보게 되었는가
4. ✅ 개발 가능성 요약 (M1~M2 압축)
5. ✅ 공급 유형과 사업 방향 (M3 요약)
6. ✅ 규모와 형태의 합리성 (M4 요약)
7. ✅ 사업 구조 한 장 설명
8. ✅ 리스크 요약
9. ✅ 그래서 무엇을 결정해야 하는가
10. ✅ Appendix (선택)

## ✅ 원안 기획 (설명용 프레젠테이션 구조)

### 보고서의 정확한 정체성

> **"이미 나온 분석 결과를 '사람이 이해하고 결정하게 만드는' 설명 전용 문서"**

**이건 뭐냐?**
- ❌ 분석 보고서 아님
- ❌ 계산 결과물 아님
- ❌ 신규 정보 생성 문서 아님
- ✅ **설득·공유·회의·설명용 프레젠테이션**

**📌 쉽게 말하면:**
"PDF 보고서를 들고 말로 설명해야 할 때 쓰는 공식 대본"

### 이 보고서가 답해야 하는 질문

1. **이 땅은 어떤 땅인가?**
2. **왜 이 사업이 가능한가?**
3. **어디까지 검증되었는가?**
4. **어디에 리스크가 있고, 어떻게 관리되는가?**
5. **그래서 지금 무엇을 결정해야 하는가?**

👉 숫자보다 **이해와 납득**이 우선

## 📄 수정 후 필수 구성 (10~15 슬라이드)

### ① Opening / 결론 먼저 (1장) 🔥 가장 중요

**포함 요소:**
- **이 사업에 대한 한 줄 정의**
  - 예: "상암 DMC 인접, LH 신축매입임대 사업 대상지"
- **현재 권장 판단:** 명확한 표시
  - 🟢 **진행 권장**
  - 🟡 **조건부 진행** (조건 명시)
  - 🔴 **중단 권장** (사유 명시)
- **"오늘 이 자리에서 무엇을 결정해야 하는가"**
  - 예: "본 사업의 본격 검토 착수 여부 결정"

**❌ 금지:**
- 종합 점수 표시 ({{ overall_score }}점)
- "{{ go_decision }}" 같은 불명확한 표현

**📌 원칙:**
결론을 숨기지 않는다. 가장 먼저 판단을 제시한다.

---

### ② 대상지 한 장 요약 (1장)

**포함 요소:**
- 위치 (주소 + 지역 특성)
- 규모 (면적, 용도지역)
- 주변 맥락 (교통, 생활권, 랜드마크)
- **지도 중심** (시각적 이해)

**예시 구조:**
```
📍 위치: 서울특별시 마포구 월드컵북로 120
📏 규모: 500㎡ (151평) | 제3종 일반주거지역
🌆 주변: 상암 DMC 인접, 홍대·연남 생활권, 지하철 6호선 월드컵경기장역 도보 5분
```

**❌ 금지:**
- 감정적 표현 ("좋은 입지", "훌륭한 위치")
- 과도한 마케팅 문구

**📌 원칙:**
맥락 전달에 집중. 말로 설명 가능한 구조.

---

### ③ 왜 이 땅을 보게 되었는가 (1장)

**포함 요소:**
- **사전 검토(E) 통과 이유**
  - 예: "법적 개발 가능, LH 매입 대상 유형 부합, 물리적 치명적 결함 없음"
- **다른 후보지 대비 특징**
  - 예: "유사 지역 대비 토지가격 경쟁력, 제도 적합성 높음"
- **초기 필터 통과 포인트**
  - 예: "GO 판단 → M2~M5 정밀 분석 진행"

**예시 구조:**
```
✓ 사전 검토 통과 (E. Quick Review)
  - 법·제도: 주택 개발 가능 (제3종 일반주거지역)
  - LH 적합성: 신축매입임대 대상 유형 부합
  - 물리적 리스크: 치명적 결함 미확인

→ "정밀 분석을 할 가치가 있다"고 판단
```

**📌 원칙:**
"왜 이걸 굳이 설명하는지"에 답한다.

---

### ④ 개발 가능성 요약 (M1~M2 압축) (1~2장)

**포함 요소:**
- **법적으로 가능한 범위**
  - 예: "건폐율 60%, 용적률 200% 이하"
- **제도상 가능한 방향**
  - 예: "LH 신축매입임대 사업 적용 가능"
- **토지의 한계와 강점**
  - 한계: "면적 500㎡로 대규모 개발 불가"
  - 강점: "정형지, 도로 접면, 경사 없음"

**예시 구조 (1장):**
```
법적 개발 가능 범위:
  - 건폐율: 60% (최대 300㎡)
  - 용적률: 200% (최대 1,000㎡)
  - 높이: 4층 이하

토지 특성:
  - ✓ 정형지 (개발 용이)
  - ✓ 도로 접면 (진입로 확보)
  - ⚠️ 면적 제약 (500㎡로 소규모)
```

**❌ 금지:**
- 숫자만 나열 ({{ floor_area_ratio }}%, {{ building_coverage_ratio }}%)
- 계산 과정 표시

**📌 원칙:**
가능 / 불가능 구조를 설명. 숫자는 최소화.

---

### ⑤ 공급 유형과 사업 방향 (M3 요약) (1~2장)

**포함 요소:**
- **왜 이 공급 유형인가**
  - 예: "지역 특성상 소형 평형 수요 높음 → 행복주택 유형 적합"
- **다른 유형은 왜 배제되었는가**
  - 예: "국민임대: 면적 부족으로 세대수 확보 어려움 (배제)"
- **정책·제도적 정합성**
  - 예: "LH 신축매입임대 행복주택 기준 충족 (입지·수요·규모)"

**예시 구조:**
```
권장 공급 유형: 행복주택

선정 이유:
  1️⃣ 지역 특성: 상암 DMC·홍대 생활권 → 소형 평형 수요 높음
  2️⃣ 정책 부합: LH 행복주택 입지 기준 충족 (대학·산업단지 인접)
  3️⃣ 경제성: 소형 평형 다세대 → 세대수 확보 유리

배제된 유형:
  ❌ 국민임대: 면적 부족 (500㎡로 세대수 확보 어려움)
  ❌ 영구임대: 입지 부적합 (신규 공급 지역 아님)
```

**📌 원칙:**
선택의 이유를 설명. "왜 이 유형인가?"에 답한다.

---

### ⑥ 규모와 형태의 합리성 (M4 요약) (1~2장)

**포함 요소:**
- **왜 이 규모인가**
  - 예: "법적 한계 용적률 200% 적용 → 총 세대수 20세대"
- **과하지 않은 이유**
  - 예: "인센티브 과다 적용 배제, 현실 시공 가능 범위 내"
- **현실적인 이유**
  - 예: "LH 매입 기준 충족, 주변 스카이라인 조화"

**예시 구조:**
```
개발 규모: 총 20세대 (소형 평형 다세대)

규모 산정 근거:
  1️⃣ 법적 한계: 용적률 200% (최대 1,000㎡)
  2️⃣ 현실 적용: 층수 4층, 세대당 평균 50㎡
  3️⃣ LH 기준: 행복주택 소형 평형 기준 충족

"최대"가 아닌 "적정" 규모:
  ✓ 인센티브 과다 적용 배제 (안정성 우선)
  ✓ 주변 건물 스카이라인 조화
  ✓ LH 매입 기준 범위 내
```

**📌 원칙:**
"최대"가 아니라 "적정"의 논리를 설명.

---

### ⑦ 사업 구조 한 장 설명 (1장) 🔥 핵심

**포함 요소:**
- **돈의 흐름 다이어그램** (필수)
  - 토지주 → 시행사 → LH
- **역할 분담**
  - 토지주: 토지 제공
  - 시행사: 개발·시공
  - LH: 매입·운영
- **Exit 구조 (LH)**
  - 예: "준공 후 LH 매입 → 시행사 수익 회수"

**예시 다이어그램 구조:**
```
[토지주]
  ↓ 토지 제공
[시행사/개발사]
  ↓ 개발·시공 (총 투자비 ○○억)
[준공]
  ↓ LH 매입 (매각가 ○○억)
[LH]
  ↓ 신축매입임대 운영
```

**❌ 금지:**
- 복잡한 재무 구조 설명
- PF 금융 로직

**📌 원칙:**
다이어그램 필수. 말로 설명 가능해야 함.

---

### ⑧ 리스크 요약 (1~2장)

**포함 요소:**
- **주요 리스크 3~4개**
  - 예: 인허가 지연, 공사비 상승, 정책 변경, 매입가 변동
- **각 리스크의 관리 가능성**
  - 예: "인허가 지연: 법적 검토 완료, 사전 협의 진행 → 관리 가능"
- **이미 제거된 리스크 vs 남아 있는 리스크**
  - 제거: "법적 개발 불가 리스크 → M1 단계에서 확인 완료"
  - 남음: "공사비 상승 → 보수적 단가 적용으로 흡수 가능"

**예시 구조:**
```
주요 리스크 및 관리 방안:

1️⃣ 인허가 지연 리스크
  - 발생 가능성: 중 (통상 6개월)
  - 관리 방안: 법적 검토 완료, 사전 협의 진행
  - 평가: ✓ 관리 가능

2️⃣ 공사비 상승 리스크
  - 발생 가능성: 중 (물가 변동)
  - 관리 방안: 보수적 단가 적용 (LH 기준 + 10%)
  - 평가: ✓ 흡수 가능

3️⃣ LH 정책 변경 리스크
  - 발생 가능성: 낮 (현행 제도 안정)
  - 관리 방안: LH 사전 협의 진행 중
  - 평가: ✓ 관리 가능

4️⃣ 매입가 변동 리스크
  - 발생 가능성: 낮 (공공 기준 적용)
  - 관리 방안: LH 매입 기준 사전 확인
  - 평가: ✓ 구조적 안정
```

**❌ 금지:**
- "문제 없다"는 식의 낙관적 표현
- 리스크 회피

**📌 원칙:**
"이 정도면 관리 가능"이라는 톤. 리스크를 숨기지 않는다.

---

### ⑨ 그래서 무엇을 결정해야 하는가 (1장) 🔥 회의용

**포함 요소:**
- **지금 결정할 것**
  - 예: "본 사업의 본격 검토 착수 여부"
- **아직 결정하지 말아야 할 것**
  - 예: "최종 매입가 확정 → LH 협의 후 결정"
- **다음 단계 액션**
  - 예: "1️⃣ LH 담당자 사전 협의, 2️⃣ 상세 실사 진행, 3️⃣ 사업 구조 확정"

**예시 구조:**
```
🎯 오늘 결정해야 할 것:

본 사업의 본격 검토 착수 여부
  → 진행 시: M2~M5 정밀 분석 결과 바탕으로 실사 착수
  → 보류 시: 추가 검토 필요 사항 명시

🔜 다음 단계 액션:

1️⃣ LH 담당자 사전 협의 (2주 내)
2️⃣ 상세 실사 및 법률 검토 (1개월)
3️⃣ 사업 구조 확정 및 투자 의사결정 (2개월)

⚠️ 아직 결정하지 말아야 할 것:

- 최종 매입가 확정 (LH 협의 후)
```

## 🎨 표현 원칙 (아주 중요)

### ❌ 하지 말아야 할 것

1. **표 과다**
   - 현재: Slide 5에서 총 투자비/예상 수익/NPV 표로 나열 ❌
   - 수정: 사업 구조 다이어그램으로 대체 ✅

2. **숫자 나열**
   - 현재: IRR {{ irr }}%, NPV {{ npv }}억 ❌
   - 수정: "사업 구조가 안정적이며, LH 매입 Exit이 명확함" ✅

3. **보고서 복붙**
   - 현재: M2~M5 결과를 그대로 슬라이드화 ❌
   - 수정: "왜?"에 대한 답 중심으로 재구성 ✅

4. **읽기용 문장**
   - 현재: 긴 문장 나열 ❌
   - 수정: 짧은 키워드 + 말로 설명 가능한 구조 ✅

### ✅ 반드시 지켜야 할 것

1. **한 슬라이드 = 한 메시지**
   - 예: Slide 1은 "결론"만, Slide 3은 "왜 이 땅인가"만 전달

2. **말로 설명 가능한 구조**
   - 숫자 대신 다이어그램
   - 표 대신 키워드 + 화살표

3. **"왜?"에 대한 답**
   - 모든 슬라이드는 "왜 이렇게 판단했는가"를 설명

4. **판단을 돕는 흐름**
   - 결론 → 맥락 → 근거 → 리스크 → 결정사항 순서
