## 1.5 AI 모델 및 알고리즘

### 1.5.1 수요 점수 산정 알고리즘

**공식**:
```
수요점수 = (입지점수 × 0.30) + (규모점수 × 0.25) + (사업성점수 × 0.30) + (법규점수 × 0.15)
```

**유형별 특화 요소**:

| 유형 | 핵심 평가 항목 | 가중치 |
|------|---------------|--------|
| 청년형 | 지하철 접근성, 대학 근접도, 청년 인구 비율 | 지하철 ≤500m: +15점 |
| 신혼·신생아 I/II | 초등학교, 어린이집, 공원 접근성 | 초등학교 ≤400m: +12점 |
| 다자녀형 | 학군, 공원, 주차 공간, 넓은 면적 | 공원 ≤500m: +10점 |
| 고령자형 | 병원, 복지시설, 저층 선호 | 병원 ≤300m: +15점 |

### 1.5.2 지리적 최적화 알고리즘

**4방향 탐색 모델**:
1. 입력 위치에서 반경 1~3km 내 후보지 탐색
2. 북/동/남/서 4방향 각 1~2개씩 선정
3. 접근성·수요·인프라·환경 4개 카테고리 점수화
4. 종합 점수 상위 3~5개 추천

**점수 산정**:
```
최적화점수 = (접근성 × 0.35) + (수요 × 0.30) + (인프라 × 0.20) + (환경 × 0.15)
```

## 1.9 재무 및 수익 모델

### 사업비 구조

```
총 사업비 = 토지비 + 건축비 + 부대비용 + 금융비용

토지비 = 토지면적 × 감정평가액
건축비 = 연면적 × 단위 공사비
부대비용 = 설계비 + 인허가비 + 부담금
금융비용 = 총 사업비 × 대출비율 × 금리 × 기간
```

### 수익 구조 (공사비 연동형 기준)

```
매입가 = 토지비 + 공사비 + 인정이익 (3~5%)

예상 수익률 = (매입가 - 총 사업비) / 총 사업비 × 100

목표 수익률: 5~8% (안정형), 8~12% (성장형)
```

## 2.2 정책 변화 상세 분석

### 2.2.1 공사비 연동형 주택 확대

**변경 내용**:
- 기존: 일부 유형(청년, 신혼부부)만 공사비 연동형 적용
- 2025년: 전 유형으로 확대 (일반형, 고령자형 포함)

**공사비 연동형 매입가 산정 방식**:
```
매입가 = 토지비 + 표준건축비 + 인정 이익

표준건축비 = 실제 공사비 (상한선 적용)
인정 이익 = 표준건축비 × 3~5%

예시:
- 토지비: 10억 원
- 표준건축비: 15억 원 (연면적 1,000㎡ × 150만원/㎡)
- 인정 이익: 15억 × 4% = 0.6억 원
→ 매입가 = 10억 + 15억 + 0.6억 = 25.6억 원
```

**ZeroSite 대응**:
- `BuildingCalculator` 모듈에 공사비 연동형 계산식 적용
- 지역별 표준건축비 데이터베이스 구축 (2024~2025 LH 공고 기준)
- 수익성 시뮬레이션 기능 추가

### 2.2.4 수요 예측 모델 고도화

**기존 vs 신규 비교**:

| 구분 | 기존 (2024) | 신규 (2025) |
|------|------------|------------|
| **평가 항목** | 인구 통계, 교통 접근성 | + 편의시설 접근성, 개발계획 반영 |
| **유형별 분석** | 통합 수요 점수 | 유형별 독립 점수 산정 |
| **가중치** | 고정 (입지 40%, 수요 60%) | 동적 조정 (입지 30~40%, 수요 25~35%) |
| **데이터 소스** | 행정안전부 인구통계 | + 국토교통부 개발계획, 교육청 학군 정보 |

**신규 수요 점수 산식**:
```
수요점수 = Σ(유형별 요소 × 가중치)

청년형:
= (지하철 점수 × 0.25) + (대학 점수 × 0.20) + (청년인구 비율 × 0.25)
  + (상업시설 점수 × 0.15) + (취업 기회 × 0.15)

신혼·신생아형:
= (초등학교 점수 × 0.25) + (어린이집 점수 × 0.20) + (공원 점수 × 0.20)
  + (병원 점수 × 0.15) + (주거환경 × 0.20)

다자녀형:
= (학군 점수 × 0.30) + (공원 점수 × 0.20) + (주차공간 × 0.15)
  + (면적 적정성 × 0.20) + (안전성 × 0.15)

고령자형:
= (병원 점수 × 0.30) + (복지시설 점수 × 0.25) + (저층 선호 × 0.20)
  + (대중교통 × 0.15) + (편의시설 × 0.10)
```

**ZeroSite 대응**:
- `_calculate_type_demand_scores()` 메서드에 신규 산식 적용
- 실시간 인구통계 API 연동 (MOIS API)
- 개발계획 데이터베이스 구축 (국토교통부 API)

## 5.2 Geo Optimization Algorithm

### 4방향 탐색 알고리즘
```python
def find_alternative_sites(lat, lng, radius_km=3):
    candidates = []
    directions = ['NORTH', 'EAST', 'SOUTH', 'WEST']
    
    for direction in directions:
        # 각 방향으로 후보지 탐색
        sites = search_direction(lat, lng, direction, radius_km)
        candidates.extend(sites[:2])  # 방향당 최대 2개
    
    # 점수 계산
    for site in candidates:
        site.score = calculate_optimization_score(site)
    
    # 상위 3~5개 반환
    return sorted(candidates, key=lambda x: x.score, reverse=True)[:5]
```

# 10. 재무 및 수익 모델

## 10.1 사업비 구조

```
총 사업비 = 토지비 + 건축비 + 부대비용 + 금융비용

토지비 = 토지면적 × 감정평가액
건축비 = 연면적 × 단위 공사비
부대비용 = 설계비 + 인허가비 + 부담금 (총 사업비의 5~8%)
금융비용 = 총 사업비 × 대출비율 × 금리 × 기간
```

## 10.2 공사비 연동형 매입가

```
매입가 = 토지비 + 표준건축비 + 인정 이익

표준건축비 = 실제 공사비 (상한선 적용)
인정 이익 = 표준건축비 × 3~5%

예시 계산:
- 토지비: 10억 원
- 표준건축비: 15억 원
- 인정 이익 (4%): 0.6억 원
→ 매입가 = 25.6억 원
```

## 10.3 수익률 계산

```
예상 수익률 = (매입가 - 총 사업비) / 총 사업비 × 100

목표 수익률:
- 안정형: 5~8%
- 성장형: 8~12%
```
