# M4 보고서 전면 재작성 계획서

## 📋 작업 개요

LH 신축매입임대 건축규모 검토 보고서(M4)를 **법규·설계·사업성·LH 심사 연결 문서** 수준으로 전면 재작성

---

## 🔴 현재 문제점 진단

### 기존 보고서의 한계
1. **세대수·주차대수 숫자만 제시**, 산정 논리 부재
2. 법정 용적률 / 인센티브 적용의 **구조적 의미 설명 부족**
3. M3(공급유형), M5(사업성), M6(LH 심사)와의 **결정 사슬 미연결**
4. "주차 0대" 같은 문제 결과를 **있는 그대로만 제시**
5. "최대 26세대 가능"이라는 **단순 결론**으로 끝남

### 핵심 문제
현재 보고서는 **"얼마나 지을 수 있는가"**만 답하고 있음  
→ 필요한 것은 **"이 사업이 성립 가능한 규모는 어디까지인가"**

---

## ✅ 9가지 요구사항 상세 분석

### 1️⃣ M4 보고서의 역할 재정의

**변경 전**:
- "이 땅에 얼마나 지을 수 있는가?"
- 법적 최대치만 계산

**변경 후**:
- "이 사업이 성립 가능한 규모는 어디까지인가?"
- 다음을 **구분하여 설명**:
  1. 법적으로 가능한 최대치
  2. LH가 수용 가능한 합리적 규모
  3. 사업성이 무너지는 임계점

**새로운 보고서 구조**:
```
I. 보고서 개요 및 M4의 역할
   - M4는 "가능 범위 계산"이 아니라 "사업 성립 가능 규모 판단"
   - M3(유형) → M4(규모) → M5(사업성) → M6(심사) 연쇄 구조
   - 최대치 추구의 위험성
```

### 2️⃣ 법·제도 기반 건축 가능 범위 분석

**변경 전**:
- 용적률 250%, 건폐율 60% → 나열만
- 규제 목록만 제시

**변경 후**:
- **"이 규제가 세대수에 어떤 영향을 주는지"** 중심 설명
- 핵심 규제 3가지만 선택하여 **세대수 임팩트** 분석

**새로운 구조**:
```
II. 법·제도 기반 건축 가능 범위 분석
   2.1 용도지역/지구 및 지구단위계획
       - 단순 나열 ❌
       - "이 규제가 있어서 세대수가 X개로 제한됨" ✅
   
   2.2 건폐율·용적률의 실제 적용 범위
       - 상한이 아니라 '실제 적용 가능한 범위'
       - 예: "법정 250%이지만, 일조권으로 인해 실제 210% 적용"
   
   2.3 핵심 제약 요인 3가지
       - 일조권 사선 → 최상층 세대 X개 감소
       - 높이 제한 → 층수 제한 → 전체 세대 Y개 감소
       - 대지 형상 → 코어 배치 제약 → 세대 효율 Z% 감소
```

### 3️⃣ 기본 시나리오 vs 인센티브 시나리오

**변경 전**:
- 단일 시나리오만 제시
- 또는 시나리오 A/B를 숫자만 비교

**변경 후**:
- **최소 2가지 시나리오** 구조화
- 각 시나리오마다 **"왜 이 정도가 한계인지"** 설명

**새로운 구조**:
```
III. 시나리오 분석: 법정 기준 vs 인센티브 적용

   시나리오 A: 법정 기준 (기본안)
   ├─ 법정 용적률 250% 적용
   ├─ 가능한 최대 연면적: 1,250㎡
   ├─ 합리적 세대수 범위: 18~22세대 (최대치 ❌ / 권장 범위 ✅)
   ├─ 주차 계획: 법정 기준 5대 vs 실제 확보 가능 3대
   └─ 한계점: "22세대 초과 시 주차 문제 발생, LH 심사 리스크"

   시나리오 B: 인센티브 적용 (확장안)
   ├─ 공공기여 용적률 인센티브 +30%p (280% 적용)
   ├─ 가능한 최대 연면적: 1,400㎡
   ├─ 합리적 세대수 범위: 22~26세대
   ├─ 인센티브 적용 조건:
   │   - LH 신축매입임대에서 실무적으로 인정 가능한 것만
   │   - "가능"이 아니라 "실무적으로 수용 가능" 기준
   └─ 한계점: "26세대 초과 시 사업성 악화 (M5에서 적자 전환)"

   시나리오 비교 및 권장안
   ├─ 시나리오 A: 안정적, 주차 리스크 관리 가능
   ├─ 시나리오 B: 세대수 증가하지만 인센티브 승인 불확실성
   └─ 권장: 시나리오 A 기준 20세대 (안전 마진 포함)
```

### 4️⃣ M3 연계 세대 구성 논리

**변경 전**:
- M3 공급유형과 무관하게 세대 구성 제시

**변경 후**:
- M3에서 결정된 **공급유형(예: 청년형)** 기준으로 세대 구성 설계

**새로운 구조**:
```
IV. M3 연계 세대 구성 논리

   4.1 M3 공급유형 확인
       - 본 사업지 공급유형: 청년형 (M3 보고서 참조)
       - 청년형 요구사항: 소형 평형, 1인 가구 중심, 최소 공용면적

   4.2 공급유형별 적정 전용면적 범위
       - 청년형: 14㎡~30㎡ (LH 기준)
       - 신혼형: 36㎡~50㎡
       - 비교: "같은 연면적 1,250㎡라도"
         * 청년형 구성 시: 20~25세대 가능
         * 신혼형 구성 시: 15~18세대 가능

   4.3 세대당 공용면적 비율
       - 복도형 vs 코어형에 따른 효율 차이
       - 청년형은 복도형이 유리 (공용면적 최소화)
       - 효율 비교:
         * 복도형: 전용률 75% → 20세대 가능
         * 코어형: 전용률 68% → 18세대 가능

   4.4 어떤 세대 구성이 더 LH 친화적인가?
       - LH 청년형 선호 평형: 16㎡, 20㎡, 25㎡
       - 권장 구성: 16㎡(40%) + 20㎡(40%) + 25㎡(20%)
       - 이유: LH 매입가 책정 용이, 임대 수요 안정
```

### 5️⃣ 주차 계획 및 LH 실무 관점 해석

**변경 전**:
- "주차 0대" → 그대로 제시하고 끝

**변경 후**:
- "주차 0대"의 **의미와 대응 방안** 명확히 설명

**새로운 구조**:
```
V. 주차 계획 및 LH 실무 관점 해석

   5.1 법정 주차 기준
       - 주차장법 기준: 세대당 0.25대 (청년형 소형 임대주택)
       - 본 사업지 적용: 20세대 × 0.25 = 5대

   5.2 청년형/소형 임대주택에서의 완화 적용 가능성
       ⚠️ "주차 0대"가 나온 이유
       - 대지 면적 500㎡ 미만 소형 부지
       - 주차장법 완화 조항 적용 가능 (제6조의2)
       - 단, LH 매입 시 최종 승인은 별도 검토 필요

   5.3 LH 매입 시 실제 문제점 vs 허용 포인트
       실제 문제가 되는 경우:
       - 신혼형/다자녀형 등 차량 수요 높은 유형
       - 대지 면적 1,000㎡ 이상 규모에서 0대 계획
       
       허용 가능한 경우 (본 사업지 해당):
       - 청년형 소형 임대 (1인 가구 중심, 차량 소유율 낮음)
       - 대중교통 접근성 우수 지역 (역세권 등)
       - 대지 면적 500㎡ 미만 소형 부지

   5.4 "주차 0대"는 리스크인가, 관리 가능한 조건인가?
       판단: 관리 가능한 조건
       이유:
       1. M3에서 청년형으로 확정 (차량 수요 낮음)
       2. 대중교통 접근성 우수 (지하철 도보 5분)
       3. 주변 공영주차장 이용 가능 (150m 거리)
       4. LH 청년형 사례에서 주차 0대 승인 前例 있음

       단, 조건:
       - 입주자 모집 시 주차 불가 명시 필수
       - 대중교통 이용 전제 조건 안내
```

### 6️⃣ M5·M6 연결 논리

**변경 전**:
- M4 보고서가 독립적으로 끝남
- 다음 모듈과의 연결 없음

**변경 후**:
- M5(사업성), M6(LH 심사)와의 **명확한 연결고리** 제시

**새로운 구조**:
```
VI. M5·M6 연결 논리 (ZeroSite 핵심)

   6.1 M5 사업성 분석과의 연계
       본 건축 규모는 M5 사업성 분석에서
       공사비/임대수익 구조상 손익분기점 이상을 확보하는 규모이며,
       
       연계 논리:
       - 권장 세대수 20세대 기준
       - 예상 공사비: 약 15억원 (세대당 7,500만원)
       - 예상 LH 매입가: 약 18억원 (세대당 9,000만원)
       - 손익분기점: 17세대 이상
       - 결론: 20세대는 손익분기점 +3세대로 안전 마진 확보
       
       과도한 최대 세대수 추구의 위험:
       - 26세대로 확장 시: 공사비 증가율 > 매입가 증가율
       - M5 분석 결과: 25세대 초과 시 적자 전환
       - 따라서 최대치 추구는 오히려 사업성 악화

   6.2 M6 LH 심사 판단과의 연계
       과도한 최대 세대수 추구는
       오히려 LH 심사(M6)에서 리스크 요인으로 작용할 수 있음
       
       LH 심사 리스크 요인:
       - 용적률 최대 활용 (250% 풀 활용): "과밀 우려"
       - 주차 0대 + 고밀도 세대 구성: "입주자 불편 예상"
       - 최소 공용면적 설계: "주거 품질 저하 우려"
       
       권장 규모의 LH 심사 유리 요인:
       - 용적률 80% 활용 (200%): "적정 밀도"
       - 세대수 20세대 (안정적): "관리 용이"
       - 공용면적 적정 확보: "주거 품질 우수"

   6.3 본 보고서의 철학
       따라서 본 보고서는
       '최대 규모'가 아닌 '통과 가능한 규모'를 제안한다
       
       근거:
       - M3: 청년형 공급유형 확정
       - M4: 법적 최대치보다 사업 가능 규모 우선
       - M5: 손익분기점 이상 확보
       - M6: LH 심사 리스크 최소화
```

### 7️⃣ 종합 판단 및 권장 건축 규모

**변경 전**:
- "최대 26세대 가능"
- 단일 숫자로 결론

**변경 후**:
- **권장 세대수 범위** 제시 (최소~최대)
- 법규·공급유형·사업성·LH 심사를 **연결한 논리형 결론**

**새로운 구조**:
```
VII. 종합 판단 및 권장 건축 규모

   7.1 최종 권장 규모
       
       LH 매입 및 사업성 관점에서
       권장 세대수 범위는 18~22세대
       
       최적 권장: 20세대

   7.2 권장 근거 (4가지 축)
       
       1. 법규 적합성
          - 법정 용적률 200% 적용 (250% 중 80%)
          - 건폐율 50% 적용 (60% 중 83%)
          - 일조권·높이 제한 모두 충족
       
       2. 공급유형 적합성 (M3 연계)
          - 청년형 공급유형에 최적화된 평형 구성
          - 16㎡(8세대) + 20㎡(8세대) + 25㎡(4세대)
          - LH 청년형 선호 평형과 일치
       
       3. 사업성 확보 (M5 연계)
          - 손익분기점 17세대 대비 +3세대 안전 마진
          - 공사비 15억 vs 매입가 18억 (이익률 20%)
          - 26세대 확장 시 오히려 적자 전환 리스크
       
       4. LH 심사 리스크 최소화 (M6 연계)
          - 적정 밀도 유지 (과밀 우려 없음)
          - 주차 0대이지만 청년형으로 관리 가능
          - 공용면적 적정 확보 (주거 품질 우수)

   7.3 18세대 vs 22세대 선택 기준
       
       18세대 (보수적 권장):
       - 주차 완화 승인 불확실 시
       - LH 심사 안정성 최우선 시
       - 공사비 리스크 최소화 필요 시
       
       22세대 (적극적 권장):
       - 주차 완화 승인 확실 시
       - 사업성 극대화 필요 시
       - M3·M5·M6 모든 조건 충족 확인 시

   7.4 최대치(26세대) 추구를 권장하지 않는 이유
       
       법적으로는 가능하지만:
       - M5: 공사비 증가로 적자 전환
       - M6: LH 심사에서 과밀 우려로 감점
       - 실무: 주차·공용면적 문제로 입주자 불편
       
       → 최대치 추구는 오히려 사업 실패 리스크 증가
```
