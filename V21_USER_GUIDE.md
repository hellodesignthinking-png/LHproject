## 🚀 빠른 시작 (Quick Start)

### **옵션 1: API 사용 (권장)**

```bash
# v21 Professional 보고서 생성
curl -X POST "http://localhost:8091/generate-report-v21" \
  -H "Content-Type: application/json" \
  -d '{
    "address": "서울특별시 강남구 역삼동 123-45",
    "land_area_sqm": 1650,
    "supply_type": "청년"
  }'

# 응답 예시:
{
  "status": "success",
  "report_url": "/reports/v21_report_20251210_160000.html",
  "generation_time": 4.8,
  "file_size_kb": 123,
  "message": "v21 Professional Report generated | 270 narrative lines, 12 policy citations | Decision: CONDITIONAL/ADOPT",
  "variables_count": 270
}
```

### **옵션 2: Python 스크립트 사용**

```bash
cd /home/user/webapp
python3 generate_v21_report.py
```

**출력**:
- `generated_reports/v21_report_YYYYMMDD_HHMMSS.html` (HTML 보고서)
- `generated_reports/v21_report_YYYYMMDD_HHMMSS.pdf` (PDF 보고서)

## 📋 v21 보고서 구조 (9개 섹션)

### **1. 경영진 요약 (Executive Summary)** - 40줄
**핵심 내용**:
- 사업 개요 (주소, 면적, 유형, 세대수)
- 주요 재무 지표 6개 (CAPEX, IRR, NPV, ROI, 회수기간, 감정평가율)
- **이중 의사결정**:
  - **재무적 판단**: PASS (IRR ≥10%) / 조건부 (IRR 8-10%) / 거부 (IRR <8%)
  - **정책적 판단**: 채택 (용도지역 적합) / 조건부 / 거부
- 전략적 제언 3-5개

**읽는 방법**:
1. 의사결정 결과 먼저 확인 (✅/⚠️/❌ 아이콘)
2. 재무 지표 테이블 검토
3. 전략적 제언 실행 계획 수립

---

### **2. 사업 개요 (Project Overview)** - 10줄
- 대상지 기본 정보 (주소, 면적, 용도지역, 주택 유형)
- 사업 규모 (계획 세대수, 건축 규모)
- LH 정책 근거 (공공주택 특별법 제2조, 제4조)

---

### **3. 용도지역 및 도시계획 (Zoning & Urban Planning)** - 30줄 ⭐
**핵심 내용**:
- 용도지역 현황 (법정 건폐율/용적률)
- **용적률 완화 조건**:
  - 법적 근거: 국토계획법 제78조, 주택법 제15조
  - 적용 완화: +30-50%p
  - 공공기여 요건: 도로 확폭, 소공원 기부채납 (대지 5-10%)
  - 예상 비용: 평당 30만원 × 대지 8%
- **교통 접근성**: 지하철 거리, 버스 노선
- **교육시설**: 학교 거리, 학교용지 부담금 (세대당 500만원)
- **LH 정책 정합성**: 입지 선정 기준 부합도

**읽는 방법**:
1. 용적률 완화율 확인 (높을수록 유리)
2. 공공기여 비용 확인 (CAPEX 반영 필수)
3. 교통/학교 점수 확인 (청년·신혼부부 선호도)

---

### **4. 수요 인텔리전스 (Demand Intelligence)** - 35줄
**핵심 내용**:
- **수요 점수 해석**:
  - 85-100점: 수요 매우 우수 (Very Strong)
  - 70-84점: 수요 우수 (Strong)
  - 50-69점: 수요 보통 (Moderate)
  - <50점: 수요 미흡 (Weak)
- 인구 구조 분석 (연령, 가구 유형, 소득 수준)
- 공급-수요 균형 (<80% 공급 부족, 80-120% 균형, >120% 과잉)
- LH 정책 부합성 (청년·신혼부부 타겟 적합도)

**읽는 방법**:
1. 수요 점수 먼저 확인 (70점 이상 권장)
2. 공급-수요 비율 확인 (85% 미만이 유리)
3. 타겟층 일치도 확인

---

### **5. 시장 인텔리전스 (Market Intelligence)** - 60줄
**핵심 내용**:
- 비교 거래 사례 3-5개 (주소, 평당 가격, 거래 시기)
- 시장 가격 포지셔닝:
  - Premium (>110% 시장가)
  - Market-Rate (90-110%)
  - Competitive (<90%)
- 가격 추세 (6-12개월)
- 정책 연계 분석 (용적률 완화 영향)

**읽는 방법**:
1. 비교 거래 테이블 확인
2. 가격 포지셔닝 확인 (Market-Rate가 안전)
3. LH 감정평가율과 비교

---

### **6. 재무 분석 (Financial Analysis)** - 70줄 ⭐⭐
**핵심 내용**:
- **CAPEX 상세 분석** (20줄):
  - 토지비 (45-55%): 평당 단가, 시장 비교
  - 건축비 (35-45%): LH 기준 ㎡당 350만원
  - 금융비용 (5-10%), 제잡비 (3-5%)
- **수익성 평가** (25줄):
  - IRR 기준: 우수(≥15%) / 양호(12-15%) / 적정(10-12%) / 조건부(8-10%) / 거부(<8%)
  - NPV: 할인 후 순이익
  - 회수기간: LH 권장 ≤2.5년
- **민감도 종합** (15줄):
  - 영향 변수 Top 3: LH 감정평가율 > 건축비 > 금리
  - Best-case: IRR +3-5%p
  - Worst-case: IRR -2-3%p
- **3단계 개선 전략** (10줄):
  1. 단기 (3개월): 감정평가 최적화 (목표 98%)
  2. 중기 (6개월): VE 원가 절감 (건축비 -5%)
  3. 장기: 정책자금 확보 (금리 -1%p)

**읽는 방법**:
1. IRR 먼저 확인 (10% 이상 목표)
2. NPV 확인 (양수여야 함)
3. 민감도 분석 확인 (주요 변수 파악)
4. 개선 전략 실행 계획 수립

---

### **7. 리스크 분석 및 완화 전략 (Risk & Strategy)** - 35줄 ⭐
**핵심 내용**:
- **리스크 매트릭스** (5개 리스크):
  - **정책 리스크**: 용적률 불허가 (HIGH/MEDIUM/LOW)
  - **재무 리스크**: IRR 목표 미달 (HIGH/MEDIUM/LOW)
  - **시장 리스크**: 임대율 하락 (MEDIUM)
  - **건축 리스크**: 공사비 증가 (MEDIUM)
  - **운영 리스크**: 유지관리비 상승 (LOW)
  - **리스크 점수**: ≤200점(승인) / 201-250(조건부) / >250(재검토)
- **완화 전략**:
  - **정책 리스크**: 사전협의, 공공기여 강화 (예방) / Plan B 설계 변경 (대응)
  - **재무 리스크**: VE 절감, 정책자금 (예방) / 사업 중단 검토 (대응)

**읽는 방법**:
1. 리스크 매트릭스에서 HIGH 항목 확인
2. 총 리스크 점수 확인 (200점 이하 권장)
3. 완화 전략 우선순위 결정

---

### **8. LH 매입 의사결정 (Government Decision Logic)** - 20줄 ⭐⭐⭐
**핵심 내용**:
- **최종 의사결정**: ✅ 매입 추천 / ⚠️ 조건부 / ❌ 매입 불가
- **판단 근거 테이블**:
  - 재무 수익성 (IRR ≥10%)
  - 순현재가치 (NPV ≥0)
  - 용도지역 적합성
  - 리스크 종합 점수 (≤200)
- **Why LH Should Buy This Property** (5가지 근거):
  1. 재무적 타당성 확보
  2. 정책 정합성 (청년·신혼부부 공급)
  3. 입지 적합성 (교통·학교)
  4. 리스크 관리 가능
  5. 사회적 ROI (+8-12%)
- **Next Steps**: 즉시 조치, 단기 (1개월), 중기 (3개월), 장기 (6개월+)

**읽는 방법**:
1. 최종 의사결정 확인 (✅/⚠️/❌)
2. 판단 근거 4개 모두 확인
3. "Why LH Should Buy" 5가지 숙지
4. Next Steps 실행 계획 수립

## 📞 FAQ (자주 묻는 질문)

### **Q1: v3와 v21 중 어느 것을 사용해야 하나요?**
**A1**:
- **빠른 초기 검토**: v3 Standard (~1초, 기본 정보)
- **정책 제출용 보고서**: v21 Professional (~5초, McKinsey급)
- **LH 본사 상신용**: v21 Professional 권장

### **Q2: IRR이 10% 미만이면 무조건 불가인가요?**
**A2**: 아닙니다.
- IRR 8-10%: 조건부 (개선 전략 실행 시 가능)
- IRR 8% 미만: 재검토 권장
- 사회적 ROI (+8-12%) 추가 고려 시 타당성 상승

### **Q3: 용적률 완화가 승인되지 않으면?**
**A3**: Plan B 실행
- 설계 변경 (세대수 조정)
- 평형 믹스 변경 (소형 → 중형)
- 재무 조정 (감정평가율 협상)

### **Q4: 리스크 점수 200점을 초과하면?**
**A4**: 완화 전략 실행
- 201-250점: 조건부 승인 (완화 전략 필수)
- 250점 초과: 재검토 권장
- 정책 리스크 vs 재무 리스크 우선순위 판단

### **Q5: 보고서 생성이 실패하면?**
**A5**: 트러블슈팅
1. API 엔드포인트 확인: `curl http://localhost:8091/health`
2. 로그 확인: `tail -f logs/zerosite.log`
3. 재시도: 1-2분 후 다시 요청
4. 지원: GitHub Issues 등록

### **Q6: 생성된 보고서를 수정할 수 있나요?**
**A6**: HTML 파일 직접 편집 가능
- 파일 위치: `generated_reports/v21_report_*.html`
- 브라우저에서 열기 → 인쇄 → PDF 저장
- 주의: 재생성 시 덮어쓰기됨

### **Q7: 여러 대상지를 한 번에 분석할 수 있나요?**
**A7**: 네, 배치 처리 가능
```python
# generate_v21_demos.py 참고
sites = [
    {"address": "강남구...", "land_area": 500},
    {"address": "마포구...", "land_area": 650},
    ...
]
for site in sites:
    generate_report(**site)
```

### **Q8: PDF 품질을 높이려면?**
**A8**: WeasyPrint 설정 조정
- 해상도: 기본 96dpi (충분)
- 폰트: Noto Sans KR (한글 최적화)
- 파일 크기: 140-160KB (적정)

## 🎓 모범 사례 (Best Practices)

### **1. 보고서 생성 전**
- ✅ 대상지 주소 정확히 확인
- ✅ 대지 면적 검증 (㎡ 단위)
- ✅ 주택 유형 선택 (청년/신혼부부/혼합)
- ✅ 용도지역 사전 확인

### **2. 보고서 검토 시**
- ✅ 경영진 요약 먼저 읽기 (30초)
- ✅ 의사결정 결과 확인 (✅/⚠️/❌)
- ✅ 재무 분석 상세 확인 (2분)
- ✅ 리스크 매트릭스 확인 (2분)

### **3. 의사결정 시**
- ✅ IRR 10% 이상 확보
- ✅ NPV 양수 확인
- ✅ 리스크 점수 200점 이하
- ✅ 정책 적합성 확인
