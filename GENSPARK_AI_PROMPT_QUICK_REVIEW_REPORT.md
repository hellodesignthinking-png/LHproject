# Genspark AI용 수정·보완 프롬프트
## E. 사전 검토 리포트 (Pre-Screening Report) - Gate 문서 전환

---

## 🎯 목표

현재 업로드된 **E. 사전 검토 리포트**를 **5분 판단 필터 Gate 문서**로 전환하여, 
"이 대상지는 정밀 분석(M2~M5)을 할 가치가 있는가?"에 대한 명확한 **GO/HOLD/STOP 판단**을 제공하는 보고서로 재구성한다.

---

## ❌ 현재 리포트의 구조적 문제 (1:1 대조 결과)

### 1. 역할 이탈: Gate 문서 → 미니 종합보고서
- **문제:** 사전 검토 리포트가 Gate 문서가 아니라 "미니 종합보고서" 형태로 출력되고 있음
- **원인:** 점수/금액/IRR 등 평가 지표가 포함되어 사전 검토 단계의 정의를 벗어남
- **영향:** 5분 판단 필터의 기획 목적 붕괴

### 2. 금지 항목 포함 (사전 검토 단계 위반)
| 항목 | 현재 출력 | 원안 기획 | 수정 방향 |
|------|-----------|-----------|-----------|
| 종합 점수 | ❌ 75점 | ⛔ 제거 | GO/HOLD/STOP 판단으로 대체 |
| IRR | ❌ 4.8% | ⛔ 제거 | 사업성 평가 단계(M5)에서만 포함 |
| 총 투자비 | ❌ 857억 | ⛔ 제거 | 사업성 평가 단계(M5)에서만 포함 |
| M2 요약 | ❌ 토지가치 ○○억 | ⛔ 제거 | M2 보고서에서만 포함 |
| M5 요약 | ❌ 수익률 IRR ○○% | ⛔ 제거 | M5 보고서에서만 포함 |
| 평가형 표현 | ❌ "공공사업 기준 적정" | ⛔ 제거 | 판단형 문장으로 대체 |

### 3. M2~M6 결과 요약 포함 (후행 보고서 침범)
- **문제:** M2(토지감정), M3(공급유형), M4(건축규모), M5(사업성), M6(종합판단) 결과가 모두 포함됨
- **원안:** 사전 검토는 "정밀 분석을 할 가치가 있는가"만 판단하고, M2~M6는 후행 보고서에서 다룸
- **수정:** M2~M6 요약 전체 제거; 법·제도 적합성 및 물리적 리스크 1차 체크만 포함

### 4. GO/STOP 판단이 흐림 (조건부 검토의 애매함)
- **문제:** "조건부 검토 권장" 같은 애매한 결론으로 GO/HOLD/STOP이 불명확
- **원안:** 반드시 **🟢 GO / 🟡 HOLD / 🔴 STOP** 중 하나를 명확히 표시
- **수정:** 명확한 판단 기준 제시 및 판단 결과를 Hero Card로 강조

### 5. "왜 더 볼 가치가 있는지/없는지" 불명확
- **문제:** 판단 근거가 산발적으로 흩어져 있고, 핵심 키워드가 명확하지 않음
- **원안:** 판단 근거 키워드 3-5개를 명확히 제시하고, 한 문장으로 결론 요약
- **수정:** Quick Decision Summary에서 판단 근거 키워드 및 한 문장 결론 명시

---

## ✅ 원안 기획 (Gate 문서 구조)

### 보고서의 단 하나의 목적
> **"이 대상지는 정밀 분석(M2~M5)을 할 가치가 있는가?"**

### 보고서 성격 강제 규정 (절대 위반 금지)
**포함 금지 항목:**
- IRR/ROI
- 총 사업비/투자비
- 종합 점수
- M2~M6 결과 수치
- 평가형 표현 ("적정", "유망", "우수" 등)

**목적:**
- 분석이 아니라 **판단 필터 문서**
- 5분 이내에 GO/HOLD/STOP 결정 가능해야 함

---

## 📄 수정 후 필수 구성 (5~7페이지)

### 1️⃣ Quick Decision Summary (1페이지)
**포함 요소:**
- **명확한 판단:** 🟢 GO / 🟡 HOLD / 🔴 STOP 중 하나
- **판단 근거 키워드:** 3-5개 (예: "법적 개발 가능", "LH 매입 대상 유형 부합", "물리적 치명적 결함 없음")
- **한 문장 결론:** 왜 GO/HOLD/STOP인지 명확히 설명

**예시 (GO 케이스):**
```
🟢 GO - 추가 분석 권장

판단 근거 키워드:
✓ 법적 개발 가능
✓ LH 매입 대상 유형 부합
✓ 물리적 치명적 결함 없음

결론 (한 문장):
본 대상지는 법·제도상 주택 개발이 가능하고, LH 매입임대 대상 유형에 부합하며, 
물리적 치명적 결함이 확인되지 않아 정밀 분석(M2~M5) 진행을 권장합니다.
```

**예시 (STOP 케이스):**
```
🔴 STOP - 추가 분석 불필요

판단 근거 키워드:
✗ 용도지역 개발 불가
✗ LH 제도상 배제 대상
✗ 치명적 물리적 결함 (경사지 30% 이상)

결론 (한 문장):
본 대상지는 용도지역상 주택 개발이 불가하며, LH 제도상 배제 대상이므로 
추가 분석을 진행하지 않습니다.
```

### 2️⃣ 부지 기본 정보 요약 (1페이지)
**포함 요소:**
- 위치/면적/용도지역
- 핵심 규제 (건폐율/용적률)
- 지도 또는 팩트 체크 박스

**톤:**
- 짧은 문장, 사실 나열, 평가 금지

### 3️⃣ 법·제도 1차 적합성 판단 (1페이지)
**포함 요소:**
- **주택 개발 가능 YES/NO**
- **LH 매입 가능성 (유형 수준만)** - 예: "{{ recommended_housing_type }} 유형 적용 가능"
- **불가 시 STOP 명시**

**판단 형식:**
```
✅ 주택 개발 가능 여부: YES - 용도지역 {{ zone_type }}에서 주택 개발 가능
✅ LH 매입 대상 가능성: 가능 ({{ recommended_housing_type }})
```

**STOP 케이스 예시:**
```
❌ 주택 개발 가능 여부: NO - 용도지역 {{ zone_type }}에서 주택 개발 불가
→ 🔴 STOP: 추가 분석 불필요
```

### 4️⃣ 물리적·입지 리스크 1차 체크 (1페이지)
**포함 요소:**
- 형상/접도/고저차/치명적 결함 여부
- **정밀 분석 제외, 결격 여부만 판단**

**체크리스트 형식:**
```
✅ 부지 형상: 양호 - 정형지 또는 개발 가능 형상
✅ 도로 접도: 양호 - 도로 접면 확보
✅ 고저차: 문제 없음 - 개발 가능 범위 내
✅ 치명적 결함: 없음 - 1차 검토에서 미확인
```

**STOP 케이스 예시:**
```
❌ 고저차: 문제 있음 - 경사지 30% 이상 (개발 불가)
→ 🔴 STOP: 추가 분석 불필요
```

### 5️⃣ 구조적 결격 사유 체크 (1페이지) 🔥 핵심
**포함 요소:**
- 제도상 배제 여부
- 해결 불가 리스크 여부
- 일정상 치명적 리스크 여부

**체크리스트 형식:**
```
✅ 제도상 배제 대상 여부: 없음 - LH 신축매입임대 사업 제도상 배제 대상 아님
✅ 구조적 해결 불가능 리스크: 없음 - 1차 검토에서 미확인
✅ 일정상 치명적 리스크: 없음 - 개발 일정에 치명적 영향 없음
```

**STOP 케이스 예시:**
```
❌ 제도상 배제 대상 여부: 있음 - LH 신축매입임대 사업 제도상 배제 대상
→ 🔴 STOP: 추가 분석 불필요
```

### 6️⃣ 사전 결론 및 다음 단계 (1페이지)
**포함 요소:**
- GO/HOLD/STOP 재확인
- GO면 다음 진행 리포트 명시 (예: "M2 토지감정평가 진행 권고")
- HOLD면 조건 명시 (예: "○○ 확인 후 재검토")
- STOP면 이유 재명시

**GO 케이스 예시:**
```
🟢 GO - 추가 분석 권장

다음 진행 권장 단계:
1. M2 토지감정평가: 정확한 토지가치 산정 및 거래사례 분석
2. M3 공급유형 판단: LH 매입임대 유형 최종 확정
3. M4 건축규모 검토: 법적·물리적 개발 가능 규모 상세 분석
4. M5 사업성 분석: 투자비, 수익 구조, IRR/NPV 분석
5. M6 LH 심사 예측: 최종 판단 및 리스크 평가
```

**STOP 케이스 예시:**
```
🔴 STOP - 추가 분석 불필요

사유:
본 대상지는 용도지역상 주택 개발이 불가하며, LH 제도상 배제 대상이므로 
추가 분석을 진행하지 않습니다.
```

## 🎨 톤 가이드

### 허용 표현
- "판단됨"
- "확인됨"
- "가능"
- "불가"
- "YES/NO"
- "GO/HOLD/STOP"

### 금지 표현
- "유망"
- "확정"
- "권장" (판단 맥락 제외)
- "적정"
- "우수"
- "긍정적"

### 문장 구조
- **짧은 문장:** 한 문장은 2줄 이내
- **판단형 문장:** "~로 판단됨", "~으로 확인됨"
- **결론 우선:** 판단 결과를 먼저 제시하고 근거는 뒤에
- **가능성/여지 표현 최소화:** "~할 수 있음" 대신 "~함"

## 🔒 최종 기준

### 보고서를 읽고 5분 이내에 다음을 결정할 수 있어야 함:
1. **이 대상지는 정밀 분석(M2~M5)을 할 가치가 있는가?** → GO/HOLD/STOP
2. **왜 그런 판단인가?** → 판단 근거 키워드 3-5개
3. **다음에 무엇을 해야 하는가?** → 다음 진행 단계 또는 종료 사유

### Gate 문서의 핵심 원칙
- **분석이 아니라 판단 필터**
- **수치가 아니라 YES/NO**
- **설명이 아니라 결론 우선**

## 🔄 6종 보고서 공통 원칙 (반드시 준수)

### 1. RUN_ID 일치 시 모든 수치 동일
- 동일한 RUN_ID를 사용하는 6종 보고서는 **모든 계산 결과가 일치**해야 함
- 예: M2의 토지가치, M5의 IRR, M6의 종합 점수는 모든 보고서에서 동일

### 2. Site Identity Block 필수
- 모든 보고서는 대상지 식별 정보를 포함해야 함
- 포함 항목: 사업명, 대상지 주소, PNU, RUN_ID, 분석 기준일

### 3. 데이터 정합성 보장
- 동일한 pipeline_result에서 데이터를 추출하여 정합성 보장
- 계산 로직 변경 금지, 표현 방식만 차별화

### 4. 계산 로직 불변 원칙
- M2~M6의 계산 로직은 절대 변경하지 않음
- 6종 보고서는 **해석 레이어만 다르게** 구성

## 📌 부연 설명

본 프롬프트는 **1:1 대조 결과**를 바탕으로, 현재 리포트의 비효율적 요소를 제거하고 **Gate 문서**로의 재구성을 위한 구체적 수정 방향을 제시합니다.

### 핵심 변경 포인트:
1. **역할 전환:** 미니 종합보고서 → Gate 문서 (5분 판단 필터)
2. **판단 명확화:** 조건부 검토 → GO/HOLD/STOP
3. **금지 항목 제거:** IRR/투자비/점수/M2~M6 요약 전체 제거
4. **판단 근거 강화:** 키워드 3-5개 + 한 문장 결론
5. **다음 단계 명시:** GO면 M2 진행 권고, STOP면 이유 재명시

### 6종 보고서 체계 내 위치:
- **E. 사전 검토 리포트:** Gate 문서 (5분 판단 필터)
