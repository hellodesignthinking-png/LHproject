## 1.3 정책 레이어 (Policy Layer)

### 1.3.1 2025년 LH 신축매입임대 정책 핵심 변화 8개

**2025년 LH 신축매입임대 사업은 다음 8가지 핵심 정책 변화를 반영하고 있습니다**:

#### **① 공사비 연동제 확대 (90% → 110%)**
- **변경 전 (2024년)**: 표준공사비의 90% 한도 내 인정
- **변경 후 (2025년)**: 표준공사비의 110% 한도로 확대 (비수도권 115%)
- **영향**: 건축비 상승 리스크 완화, 사업성 개선 (ROI +2~3%p 예상)
- **ZeroSite 대응**: 공사비 연동형 ROI 계산 엔진 업데이트 완료

#### **② 착공 심사 강화 (착공 후 6개월 → 3개월 이내 심사)**
- **변경 전**: 착공 후 6개월 이내 진도 점검
- **변경 후**: 착공 후 3개월 이내 1차 심사 + 진도율 20% 확인 필수
- **영향**: PM 일정 관리 강화 필요, 지연 시 약정 취소 위험
- **ZeroSite 대응**: PM Flow 자동 알림 기능 (착공일 기준 D+90 알림)

#### **③ ESG 평가 신설 (45점 배점)**
- **신규 항목**: 에너지효율(20점), 녹색건축인증(15점), 탄소저감(10점)
- **기존 총점**: 305점 → **신규 총점**: 350점
- **영향**: G-SEED 우수 등급 취득 시 +10점, ZEB 5등급 +3점
- **ZeroSite 대응**: ESG 자동 평가 모듈 탑재 (K-ESG 80점 이상 달성 전략)

#### **④ 유형별 수요점수 차별화 (청년·신혼 중심 강화)**
- **청년 유형**: 역세권 접근성 가중치 0.35 → 0.40 상향
- **신혼·신생아 I**: 초등학교 접근성 필수 기준 강화 (500m → 300m)
- **다자녀·고령자**: 커뮤니티 시설 의무 기준 신설 (5㎡/세대 이상)
- **ZeroSite 대응**: Type-specific Demand Scoring 알고리즘 업데이트

#### **⑤ 다필지 통합 개발 인센티브 (Cluster Synergy +5점)**
- **신규 가점**: 3필지 이상 통합 개발 시 +5점 (기존 0점)
- **조건**: 총 대지면적 1,000㎡ 이상 + 세대수 25세대 이상
- **영향**: 소규모 필지 통합 전략 필요성 증가
- **ZeroSite 대응**: Parcel Cluster Analyzer 시너지 자동 계산

#### **⑥ 감정평가 기준 변경 (공시지가 → 실거래가 반영 비중 확대)**
- **변경 전**: 공시지가 80% + 실거래가 20%
- **변경 후**: 공시지가 60% + 실거래가 40%
- **영향**: 실거래가 높은 지역(강남·송파 등) 감정가 상승
- **ZeroSite 대응**: 실거래가 기반 매입가 예측 모델 보강

#### **⑦ 임대료 산정 기준 세분화 (유형별 차등 적용)**
- **청년**: 시세 50% (변동 없음)
- **신혼·신생아 I**: 시세 60% (변동 없음)
- **신혼·신생아 II**: 시세 70% → 65% 인하
- **다자녀**: 시세 70% → 60% 인하
- **영향**: 다자녀 유형 수익성 개선 (임대료 하락 → LH 매입가 상승)

#### **⑧ 비수도권 가점 확대 (지역균형 발전)**
- **비수도권 가점**: +5점 → +8점으로 상향
- **대상 지역**: 대구, 광주, 대전, 울산, 세종, 기타 중소도시
- **영향**: 비수도권 사업 매력도 증가 (ROI +1~2%p)
- **ZeroSite 대응**: 지역별 가점 자동 반영 알고리즘

```python
def calculate_buffer_analysis(parcel_coords, radius=500):
    """
    500m 반경 내 POI 분석 및 유형별 수요점수 계산
    """
    poi_list = fetch_poi_from_kakao_map(parcel_coords, radius)
    
    type_scores = {
        "청년": 0,
        "신혼·신생아 I": 0,
        "다자녀": 0,
        "고령자": 0
    }
    
    for poi in poi_list:
        distance = calculate_distance(parcel_coords, poi.coords)
        if distance <= radius:
            # 유형별 가중치 적용
            if poi.category == "지하철역":
                type_scores["청년"] += 18.5 * (1 - distance/radius)
                type_scores["신혼·신생아 I"] += 15.0 * (1 - distance/radius)
            elif poi.category == "초등학교":
                type_scores["신혼·신생아 I"] += 20.0 * (1 - distance/radius)
                type_scores["다자녀"] += 15.0 * (1 - distance/radius)
            elif poi.category == "병원":
                type_scores["고령자"] += 18.0 * (1 - distance/radius)
    
    return type_scores
```

```python
def analyze_parcel_cluster(parcel_list):
    """
    다필지 통합 개발 시너지 자동 계산
    """
    center_parcel = parcel_list[0]  # 기준 필지
    
    cluster_candidates = []
    for parcel in parcel_list[1:]:
        distance = calculate_distance(center_parcel.coords, parcel.coords)
        if distance <= 150:  # 150m 이내만 통합 가능
            cluster_candidates.append({
                "parcel": parcel,
                "distance": distance,
                "synergy_score": calculate_synergy(center_parcel, parcel)
            })
    
    # LH 가점 조건 확인
    total_area = sum([p["parcel"].area for p in cluster_candidates]) + center_parcel.area
    total_households = sum([p["parcel"].households for p in cluster_candidates]) + center_parcel.households
    
    if len(cluster_candidates) >= 2 and total_area >= 1000 and total_households >= 25:
        lh_bonus = 5  # LH 다필지 가점 +5점
        roi_improvement = 1.8  # ROI +1.8%p
    else:
        lh_bonus = 0
        roi_improvement = 0
    
    return {
        "cluster_candidates": cluster_candidates,
        "total_area": total_area,
        "total_households": total_households,
        "lh_bonus": lh_bonus,
        "roi_improvement": roi_improvement
```

### 5.6.3 기술 구현 (Python Pseudocode)

```python
def geo_optimizer(input_parcel, radius=3000):
    """
    3km 반경 내 대안 후보지 3~5개 자동 추천
    """
    if input_parcel.lh_score >= 250:
        return {"status": "PASS", "alternatives": []}
    
    # Vworld API로 3km 반경 필지 검색
    candidate_parcels = fetch_nearby_parcels(input_parcel.coords, radius)
    
    alternatives = []
    for parcel in candidate_parcels:
        # 각 필지에 대해 ZeroSite 분석 실행
        analysis_result = run_full_analysis(parcel)
        
        if analysis_result.lh_score >= 250:
            alternatives.append({
                "parcel": parcel,
                "distance": calculate_distance(input_parcel.coords, parcel.coords),
                "lh_score": analysis_result.lh_score,
                "roi": analysis_result.roi,
                "best_type": analysis_result.best_type,
                "key_strengths": analysis_result.strengths,
                "risks": analysis_result.risks
            })
    
    # ROI 기준 상위 3개 정렬
    alternatives.sort(key=lambda x: x["roi"], reverse=True)
    
    return {
        "status": "ALTERNATIVES_FOUND",
        "alternatives": alternatives[:3],
        "recommendation": alternatives[0] if alternatives else None
    }
```

**ROI 계산식**:

```
총 사업비 = 토지비 + 건축비 + 부대비용 + 금융비용
LH 매입가 = MAX(감정가, 공사비 연동가)
```

### 9.3.3 ZeroSite 자동 추천 로직 (Python Pseudocode)

```python
def recommend_zeb_strategy(project_budget, target_roi):
    """
    프로젝트 예산 및 목표 ROI에 따라 최적 ZEB 등급 자동 추천
    """
    zeb_scenarios = [
        {"grade": 5, "cost_per_sqm": 100000, "lh_points": 3, "roi_impact": -0.3},
        {"grade": 3, "cost_per_sqm": 400000, "lh_points": 10, "roi_impact": 0.5},
        {"grade": 1, "cost_per_sqm": 1000000, "lh_points": 20, "roi_impact": -1.5}
    ]
    
    recommended = None
    for scenario in zeb_scenarios:
        total_cost = scenario["cost_per_sqm"] * project_area
        expected_roi = base_roi + scenario["roi_impact"]
        
        if expected_roi >= target_roi and total_cost <= project_budget:
            if recommended is None or scenario["lh_points"] > recommended["lh_points"]:
                recommended = scenario
    
    return recommended or zeb_scenarios[0]  # 최소 비용 전략 기본 반환
```

### 9.5.2 탄소 배출량 계산 (ZeroSite 자동 LCA 모듈)

#### 건축물 전 생애주기 탄소 배출량 산정식

```
총 탄소 배출량 (tCO2eq) = E_construction + E_operation + E_end_of_life

여기서:
E_construction = 건설 단계 탄소 배출량 (자재 생산 + 운송 + 시공)
E_operation = 운영 단계 탄소 배출량 (냉난방 + 조명 + 급탕)
E_end_of_life = 폐기 단계 탄소 배출량 (철거 + 폐기물 처리)
```

#### Python 자동 계산 예시

```python
def calculate_lca_carbon_emissions(building_data):
    """
    건축물 전 생애주기 탄소 배출량 자동 계산
    """
    # 건설 단계 (자재별 탄소 배출 계수)
    E_construction = (
        building_data["concrete_volume"] * 0.245 +  # 콘크리트: 0.245 tCO2eq/m³
        building_data["steel_weight"] * 2.5 +       # 철근: 2.5 tCO2eq/ton
        building_data["glass_area"] * 0.12 +        # 유리: 0.12 tCO2eq/m²
        building_data["insulation_volume"] * 0.08   # 단열재: 0.08 tCO2eq/m³
    )
    
    # 운영 단계 (30년 기준, 에너지 소비량 × 탄소 배출 계수)
    annual_energy_consumption = (
        building_data["heating_kwh"] +
        building_data["cooling_kwh"] +
        building_data["lighting_kwh"] +
        building_data["hot_water_kwh"]
    )
    E_operation = annual_energy_consumption * 0.466 * 30  # 0.466 kgCO2eq/kWh × 30년
    
    # 폐기 단계 (건설 단계의 5% 추정)
    E_end_of_life = E_construction * 0.05
    
    total_emissions = E_construction + E_operation + E_end_of_life
    
    # 기준 건물 대비 감축률 계산
    baseline_emissions = 1200  # 기준 건물 (tCO2eq, 예시)
    reduction_rate = (baseline_emissions - total_emissions) / baseline_emissions * 100
    
    return {
        "total_emissions": total_emissions,
        "reduction_rate": reduction_rate,
        "lh_points": get_lh_points_from_reduction(reduction_rate)
    }

def get_lh_points_from_reduction(reduction_rate):
    """
    탄소 감축률에 따른 LH 배점 자동 계산
    """
    if reduction_rate >= 30:
        return 10
    elif reduction_rate >= 20:
        return 7
    elif reduction_rate >= 10:
        return 4
    else:
        return 0
```

## 10.3 IRR (Internal Rate of Return) 분석

### 10.3.1 IRR 계산식

```
IRR = r (단, NPV = 0을 만족하는 할인율 r)

NPV = ∑[CF_t / (1+r)^t] = 0

여기서:
CF_t = t 시점의 현금흐름
r = 내부수익률 (IRR)
t = 시간 (개월)
```

### 10.3.2 IRR 계산 예시 (Python)

```python
import numpy as np

def calculate_irr(cash_flows):
    """
    현금흐름 기반 IRR 자동 계산
    """
    return np.irr(cash_flows) * 12  # 연간 IRR로 변환

# 현금흐름 예시 (18개월 프로젝트)
cash_flows = [
    -30,  # 0개월: 자기자본 투입 -30억원
```

## 10.4 NPV (Net Present Value) 분석

### 10.4.1 NPV 계산식

```
NPV = ∑[CF_t / (1+WACC)^t] - 초기 투자액

여기서:
WACC = 가중평균자본비용 (Weighted Average Cost of Capital)
WACC = (E/V) × Re + (D/V) × Rd × (1-Tc)

E = 자기자본 (30억원)
D = 타인자본 (70억원, PF+HUG)
V = 총 자본 (100억원)
Re = 자기자본비용 (12% 가정)
Rd = 타인자본비용 (4%)
Tc = 법인세율 (22%)
```

### 10.4.2 WACC 계산

```
WACC = (30/100) × 12% + (70/100) × 4% × (1-0.22)
     = 3.6% + 2.184%
     = 5.784% ≈ 5.8%
```

### 10.4.3 NPV 계산 예시

```python
def calculate_npv(cash_flows, wacc):
    """
    현금흐름 및 WACC 기반 NPV 자동 계산
    """
    npv = 0
    for t, cf in enumerate(cash_flows):
        npv += cf / ((1 + wacc) ** (t / 12))  # 월단위 → 연단위 변환
    return npv

wacc = 0.058
cash_flows = [-30, 0, 0, ..., +46]
npv = calculate_npv(cash_flows, wacc)
print(f"NPV: {npv:.2f}억원")  # 예상: +12.5억원
```

| **기준 (Base)** | 5.8% | 8억원 | **+12.5억원** | 목표 달성 ✅ |

| 29 | **WACC (가중평균자본비용)** | Weighted Average Cost of Capital | 자기자본비용과 타인자본비용을 가중 평균한 기업의 자본 조달 비용 |

| 31 | **CapEx (자본적 지출)** | Capital Expenditure | 토지 매입비, 건축비, 부대비용 등 프로젝트 초기 투자 비용 |
| 32 | **OpEx (운영 비용)** | Operating Expense | 건물 운영 단계에서 발생하는 유지보수, 관리, 임대료 등 비용 (LH 사업은 OpEx 불필요) |
