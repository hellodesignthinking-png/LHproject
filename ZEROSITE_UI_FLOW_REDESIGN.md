# 🎨 ZeroSite UI Flow Redesign - 감정평가 우선 프로세스

**목적**: 사용자 경험을 "토지감정평가가 모든 분석의 시작점"이라는 개념에 맞춰 재설계

**작성일**: 2025-12-14  
**대상**: UI/UX 디자이너, Frontend 개발자

---

## 🔥 핵심 개념

### **"토지감정평가가 모든 엔진의 Single Source of Truth"**

사용자가 주소를 입력하면:
1. **먼저 토지감정평가가 완료**됨
2. 그 결과를 기반으로 다른 모든 탭의 내용이 채워짐
3. 사용자는 "하나의 일관된 스토리"를 보게 됨

---

## 📌 전체 흐름 요약 (Top Level Flow)

```
┌──────────────────┐
│  [사용자 입력]     │
│  • 주소 (필수)     │
│  • 면적 (필수)     │
│  • 특성 (선택)     │
└────────┬─────────┘
         ↓
    [🚀 분석 실행]
         ↓
┌──────────────────┐
│ [1] 토지감정평가   │ ← 가장 먼저 (30초)
│  (v39 기준엔진)    │
└────────┬─────────┘
         ↓
┌──────────────────┐
│ [2] 토지진단       │ ← 감정평가 결과 기반 (즉시)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ [3] 규모검토       │ ← 감정평가 zoning 사용 (즉시)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ [4] 시나리오 A/B/C │ ← 감정평가 가치 기반 (즉시)
└────────┬─────────┘
         ↓
┌──────────────────┐
│ [5] 종합 보고서    │ ← 전체 통합 (즉시)
└──────────────────┘
```

**핵심**: 감정평가 이후의 모든 탭은 **즉시** 표시됨 (재계산 없음)

```
│  ⏳ [4/4] 최종 보고서 생성 대기                      │
```

**진행 단계** (4단계):
1. **토지 기본 정보 조회** (10초) - 주소 검색, 좌표 변환
2. **감정평가 실행** (15초) - v39 엔진 실행 (가장 중요)
3. **시나리오 비교 분석** (3초) - A/B/C 계산
4. **최종 보고서 생성** (2초) - Context 저장

## 🎨 UI/UX 원칙

### **1. 일관성 (Consistency)**
```
같은 데이터는 어느 탭에서 보든 100% 동일
용도지역 = 제2종일반주거지역 (모든 탭에서 동일)
```

### **2. 명확성 (Clarity)**
```
"이 데이터는 감정평가 결과입니다" 라벨 표시
데이터 출처 명시: "감정평가 기반", "실시간 조회"
```

### **3. 순서 논리성 (Logical Flow)**
