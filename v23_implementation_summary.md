### 2. Dynamic CAPEX 계산 모듈 구현 ✅
**파일**: `app/services_v13/dynamic_capex_calculator.py` (512줄)

#### 기능

##### 2-1. 시장가 기반 토지비 동적 계산
```
입력:
  - 시장 토지가: 242.00억원 (국토부 실거래가 API)
  - 협상 할인율: 5% (기본값)
  - CAPEX 상한: 50% (기본값)

계산:
  - 협상 후 토지가: 229.90억원 (242억 × 0.95)
  - CAPEX 토지비: 150.00억원 (300억 × 0.50)
  - 기존 토지비: 75.00억원 (300억 × 0.25, 고정)

결과:
  ⚠️ CAPEX 증액 필요
  - 이유: 실제 토지비(229.90억)가 CAPEX 토지비(150억)보다 79.90억 부족
  - 현재 CAPEX: 300억원
  - 권장 CAPEX: 460억원 (+160억원)
```

##### 2-2. LH 인정 건축비 검증
```
기준:
  - LH 표준건축비: 350만원/㎡
  - LH 인정 한도: 420만원/㎡ (표준 +20%)

현재 CAPEX 건축비:
  - 직접 건축비: 165억원 (CAPEX의 55%)
  - 연면적: 2,200㎡
  - 단가: 750.0만원/㎡

검증 결과:
  ❌ ERROR - 건축비 과다
  - 초과액: 400.0만원/㎡
  - 초과율: 114.3%
  
위험:
  ⚠️ LH 매입가 < CAPEX 위험 (사업성 없음)
  
권장:
  - 건축비를 420만원/㎡ 이하로 조정
  - 또는 CAPEX 구조 재조정
```

### 3. 국토부 실거래가 API 통합 ✅
**파일**: `app/services_v13/land_trade_api.py` (425줄)

#### 3-Layer Land Valuation

```
Layer 1: Market Value (국토부 실거래가)
  ├─ 실거래가 데이터 수집
  ├─ 평균 단가 계산
  ├─ 신뢰도 평가 (HIGH/MEDIUM/LOW)
  └─ 지역 필터링 (반경 500m~2km)

Layer 2: LH Appraisal (감정평가)
  ├─ 토지 감정가: 시장가 × 감정평가율 (92%)
  ├─ 건물 감정가: LH 표준단가 × 연면적
  └─ 총 LH 매입가: 토지 + 건물 감정가

Layer 3: CAPEX Fallback (보수적 추정)
  ├─ 실거래가 부족 시 작동
  ├─ CAPEX 기반 토지비 추정
  └─ 신뢰도: FALLBACK
```

#### 데이터 신뢰도 기준

| 등급 | 실거래 건수 | 설명 |
|------|-----------|------|
| HIGH | 10건 이상 | 통계적으로 신뢰 가능 |
| MEDIUM | 3-9건 | 참고 가능, 보수적 해석 필요 |
| LOW | 1-2건 | 단순 참고용 |
| FALLBACK | 0건 | CAPEX 기반 추정 (보수적) |
