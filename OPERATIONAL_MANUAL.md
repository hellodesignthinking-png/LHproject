## 🏢 1. 운영 시나리오

### 1.1 LH 신축매입임대 사업 검토 (표준 시나리오)

**담당자**: 토지개발팀 / 사업기획팀  
**소요 시간**: 최초 분석 5분 + 보고서 검토 30분 = **총 35분**  
**기존 방식 대비**: 기존 3일 → ZeroSite 35분 (**약 90% 시간 단축**)

#### STEP-by-STEP 운영 절차

```
┌─────────────────────────────────────────────────┐
│ 1. 토지 정보 확인 (담당자가 준비)                │
└─────────────────────────────────────────────────┘
  준비물:
  - 주소 또는 지번
  - 토지대장 PDF (선택)
  - 등기부등본 PDF (선택)

┌─────────────────────────────────────────────────┐
│ 2. ZeroSite 접속 및 M1 입력 (5분)               │
└─────────────────────────────────────────────────┘
  URL: http://localhost:3000/pipeline
  또는: https://zerosite.lhproject.kr/pipeline

  STEP 0: 시작
  STEP 1: 주소 검색 → 행정안전부 API 자동 조회
  STEP 2: 위치 확인 → 카카오/네이버 지도 API
  STEP 3: 지적정보 → API 우선, 없으면 PDF 또는 수기
  STEP 4: 법적정보 → 용도지역, FAR, BCR, 높이제한
  STEP 5: 도로정보 → 접도 폭, 유형
  STEP 6: 시장정보 → 공시지가, 거래사례
  STEP 7: 검토 → 입력 내용 확인
  STEP 8: M1 확정 → "분석 시작" 버튼 클릭

  ✅ 자동 저장: Redis (24시간) + DB (영구)

┌─────────────────────────────────────────────────┐
│ 3. 자동 분석 실행 (30초)                        │
└─────────────────────────────────────────────────┘
  M2: 감정평가 (5초)
    - 거래사례 수집
    - 4-Factor Premium 계산
    - 신뢰도 점수 계산
    - ⚠️ Warning 자동 생성 (거래사례 < 3건)

  M3: 주택유형 선택 (5초)
    - 청년형/신혼형/다자녀형/고령자형 점수 계산
    - 입지 분석 (역세권, 학교 등)
    - ⚠️ Tie 감지 (점수 차이 < 5점)

  M4: 규모 분석 (10초)
    - Legal FAR vs Incentive FAR
    - Alternative A (자주식 주차) vs Alternative B (기계식 주차)
    - 세대수, 연면적, 주차대수 계산

  M5: 사업성 분석 (5초)
    - NPV, IRR, ROI 계산
    - 사업성 등급 (A/B/C/D/F)
    - Base Year 명시 (공사비 2025년 기준)

  M6: LH 심사 예측 (5초)
    - 총 100점 기준 점수 계산
    - GO / 조건부 GO / NO-GO 판정
    - 개선사항 제안

┌─────────────────────────────────────────────────┐
│ 4. 결과 검토 (20분)                             │
└─────────────────────────────────────────────────┘
  화면 확인:
  - M2: 감정평가 신뢰도 A/B/C/D 등급 확인
  - M3: 주택유형 추천 (Tie 시 1순위/2순위 병기)
  - M4: Alternative A/B 비교표 검토
  - M5: ROI 18.5% → 사업성 B등급
  - M6: LH 심사 85점 → GO 판정

  체크리스트:
  - [ ] 감정평가 신뢰도 B 이상인가?
  - [ ] 거래사례 Warning이 있는가?
  - [ ] Alternative B (인센티브)가 현실적인가?
  - [ ] 사업성 등급이 C 이상인가?
  - [ ] LH 심사 점수가 70점 이상인가?

┌─────────────────────────────────────────────────┐
│ 5. 보고서 생성 (5분)                            │
└─────────────────────────────────────────────────┘
  6종 보고서:
  1. 사전검토보고서 (15p) → LH 제출용
  2. 감정평가서 (20p) → 내부 검토용
  3. LH심사예측 (10p) → 의사결정용
  4. 사업성분석 (15p) → 재무팀 공유
  5. 종합보고서 (60p) → 전문가용
  6. 요약보고서 (3p) → 토지주 설명용

  ✅ 데이터 품질 요약 페이지 포함 (표지 다음)
  ✅ Base Year 명시 (2025년 기준)
  ✅ 면책 조항 포함

┌─────────────────────────────────────────────────┐
│ 6. 의사결정 (5분)                               │
└─────────────────────────────────────────────────┘
  GO 조건:
  - 감정평가 신뢰도 B 이상
  - 사업성 등급 C 이상
  - LH 심사 점수 70점 이상

  조건부 GO:
  - 거래사례 추가 확보 후 재분석
  - Alternative B 인센티브 조건 재확인
  - 주차 대안 변경 검토

  NO-GO:
  - 감정평가 신뢰도 D
  - 사업성 등급 F
  - LH 심사 점수 60점 미만

┌─────────────────────────────────────────────────┐
│ 7. LH 제출 (담당자 업무)                        │
└─────────────────────────────────────────────────┘
  제출 서류:
  - 사전검토보고서 (ZeroSite 생성)
  - 토지대장, 등기부등본 (원본)
  - 지적도, 건축물대장 (해당 시)
  - 사업계획서 (자체 작성)

  ✅ ZeroSite 보고서는 "참고자료"로 첨부
  ✅ 데이터 출처 투명 공개 (신뢰도 향상)
```

### 1.2 다중 필지 비교 분석 (고급 시나리오)

**담당자**: 사업개발팀장  
**소요 시간**: 필지당 5분 × 3필지 = **15분**

**절차**:
1. 필지 A 분석 → Context ID 저장
2. 필지 B 분석 → Context ID 저장
3. 필지 C 분석 → Context ID 저장
4. 비교 대시보드에서 3개 필지 동시 비교
5. ROI, LH 점수, 사업성 등급 일괄 비교
6. 최적 필지 선택

**출력**: 3필지 비교표 + 추천 필지 근거

---

### 1.3 거래사례 부족 시 대응 시나리오

**상황**: M2 분석 후 "거래사례 2건: 신뢰도 저하 가능" Warning 발생

**대응 절차**:
```
1. Warning 확인
   "거래사례 부족 (2건): 신뢰도 저하 가능"
   "추가 거래사례 확보 권장"

2. 선택지
   Option A: 추가 거래사례 수기 입력
     → M1 STEP 6으로 이동
     → "거래사례 추가" 버튼
     → 부동산 거래 사이트에서 복사/붙여넣기
     → 재분석

   Option B: 보수적 평가로 진행
     → "그래도 계속" 선택
     → 신뢰도 C 등급 적용
     → 감정가에 안전 마진 -10% 적용
     → 보고서에 Warning 명시

   Option C: 분석 중단
     → "중단" 선택
     → M1으로 복귀
     → 다른 필지 선택
```

**권장**: Option A (추가 입력) 우선, 불가능 시 Option B

## 💾 3. 데이터 보관 정책

### 3.1 데이터 계층 구조

```
┌─────────────────────────────────────────────┐
│ L1: HOT (Redis) - 24시간                    │
│ - 빠른 조회 (< 10ms)                        │
│ - Context 임시 저장                         │
│ - TTL 24시간 자동 만료                      │
└─────────────────────────────────────────────┘
              ↓ (동시 dual-write)
┌─────────────────────────────────────────────┐
│ L2: WARM (PostgreSQL/SQLite) - 영구         │
│ - 안정적 백업 (< 100ms)                     │
│ - context_snapshots 테이블                  │
│ - Redis 장애 시 자동 복구                   │
│ - 접근 횟수 추적 (access_count)            │
└─────────────────────────────────────────────┘
              ↓ (주기적 아카이브)
┌─────────────────────────────────────────────┐
│ L3: COLD (S3/Object Storage) - 6개월+      │
│ - 장기 보관 (6개월 이상 미접근)             │
│ - 압축 저장 (gzip)                          │
│ - 법적 보존 의무 (3년)                      │
└─────────────────────────────────────────────┘
```

---

### 3.2 데이터 보관 기간

| 데이터 유형 | 보관 기간 | 저장 위치 | 삭제 정책 |
|------------|----------|----------|----------|
| Context (M1) | 영구 | L2 (DB) | 수동 삭제만 |
| Analysis (M2-M6) | 영구 | L2 (DB) | 수동 삭제만 |
| Reports (PDF) | 3년 | L3 (S3) | 3년 후 자동 삭제 |
| Logs | 90일 | L2 (DB) | 90일 후 자동 삭제 |
| Redis Cache | 24시간 | L1 (Redis) | 24h TTL 자동 |
| Backup | 30일 | L2 (DB) | 30일 rolling |

### 3.3 개인정보 처리

**수집하는 개인정보**: ❌ **없음**

ZeroSite는 **토지 정보만** 처리하며, 개인정보를 수집하지 않습니다:
- ❌ 토지주 성명, 주민등록번호
- ❌ 연락처, 이메일
- ❌ 금융 정보

**수집하는 정보**:
- ✅ 토지 주소, 지번
- ✅ 지적 정보 (면적, 지목, 용도지역)
- ✅ 시장 정보 (공시지가, 거래사례)
- ✅ 분석 결과 (감정가, 사업성 등)

**GDPR/개인정보보호법 준수**: ✅ **해당 없음** (개인정보 미수집)

---

### 3.4 데이터 삭제 요청 처리

**사용자 요청 시 삭제 절차**:
```sql
-- 1. Context 삭제
DELETE FROM context_snapshots WHERE context_id = '{context_id}';

-- 2. Redis 삭제
REDIS> DEL context:{context_id}

-- 3. 보고서 삭제 (S3)
aws s3 rm s3://zerosite-reports/{context_id}/

-- 4. 로그 삭제
DELETE FROM analysis_logs WHERE context_id = '{context_id}';
```

**소요 시간**: 즉시 (< 1분)  
**복구 가능 여부**: ❌ 복구 불가 (영구 삭제)

## ⚖️ 4. 법적 책임 및 면책 조항

### 4.1 시스템 역할 정의

**ZeroSite는**:
- ✅ **의사결정 지원 도구** (Decision Support System)
- ✅ **참고자료 생성** (Reference Material Generator)
- ✅ **데이터 분석 자동화** (Data Analysis Automation)

**ZeroSite는 아닙니다**:
- ❌ **감정평가사** (공인 감정평가 아님)
- ❌ **법적 자문** (법률 검토 아님)
- ❌ **LH 최종 판정** (LH 심사 결과 아님)
