
**현재 상태**: ✅ 정상 작동

### ✅ 2. API 실패 / 데이터 부족 시나리오

#### 2.1 주소 검색 API 실패
**시나리오**: 행정안전부 API가 응답하지 않음

**현재 동작**: ❌ 에러 메시지만 표시
```
"주소 검색 실패: 네트워크 오류"
```

**권장 개선**:
```javascript
// 사용자 친화적 에러 처리
if (apiError === "NETWORK_ERROR") {
  showMessage({
    type: "error",
    title: "주소 검색 일시 중단",
    message: "행정안전부 API가 일시적으로 응답하지 않습니다.",
    actions: [
      {label: "재시도", onClick: retrySearch},
      {label: "수기 입력", onClick: manualInput}
    ],
    note: "수기 입력 시 데이터 출처가 'MANUAL'로 기록됩니다."
  });
}
```

**우선순위**: 🔴 HIGH  
**예상 시간**: 1.5시간

### ✅ 3. 전문가용이지만 부담스럽지 않은 흐름

#### 3.1 용어 설명 (Glossary)
**현재 상태**: ❌ 전문 용어가 설명 없이 사용됨

**권장 개선**:
```
[추가] 모든 전문 용어에 툴팁 제공
  - FAR (Floor Area Ratio): 용적률 - 대지면적 대비 건축물 연면적 비율
  - BCR (Building Coverage Ratio): 건폐율 - 대지면적 대비 건축면적 비율
  - NPV (Net Present Value): 순현재가치 - 투자 수익성 지표
  - IRR (Internal Rate of Return): 내부수익률 - 투자 수익률

[추가] "전문가 모드" vs "간편 모드" 토글
  간편 모드: 핵심 정보만 표시, 전문 용어 최소화
  전문가 모드: 모든 상세 정보 표시
```

**우선순위**: 🟡 MEDIUM  
**예상 시간**: 1.5시간

#### 3.2 M4 Alternative A/B 비교 화면
**현재 상태**: ✅ 정상 작동 (Legal vs Incentive FAR)

**권장 개선**:
```
[추가] 비교표 상단에 설명 문구
  "Alternative A: 법적 최대 규모 (기본 용적률)"
  "Alternative B: 인센티브 규모 (LH 우대 용적률)"

[추가] 각 대안의 장단점 명시
  Alt A: ✅ 보수적, ✅ 승인 용이, ❌ 수익성 낮음
  Alt B: ✅ 수익성 높음, ⚠️ 인센티브 조건 충족 필요

[개선] 주차 대안 표시 방식
  현재: "Alt A: Self-parking / Alt B: Mechanical"
  개선: "Alt A: 자주식 주차 (램프 필요) / Alt B: 기계식 주차 (추가 비용)"
```

**우선순위**: 🟢 LOW

#### 3.3 M6 GO/NO-GO 결정 화면
**현재 상태**: ✅ 정상 작동

**권장 개선**:
```
[추가] 결정 근거 요약
  ✅ GO 권장:
    - 감정평가 신뢰도: A (0.87)
    - 사업성 등급: B (ROI 17.5%)
    - LH 심사 예측: 통과 가능성 높음 (85점)

  ⚠️ 조건부 GO:
    - 감정평가 신뢰도: B (0.75)
    - 거래사례 추가 확보 필요
    - 주차 대안 B 검토 권장

  ❌ NO-GO:
    - 사업성 등급: D (ROI 3.2%)
    - LH 심사 예측: 통과 어려움 (55점)
    - 입지 개선 후 재검토 권장

[추가] 다음 단계 안내
  ✅ GO → "6종 보고서 생성하기"
  ⚠️ 조건부 GO → "개선사항 검토 후 보고서 생성"
  ❌ NO-GO → "M1으로 돌아가기 / 다른 필지 분석"
```

**우선순위**: 🟡 MEDIUM  
**예상 시간**: 1시간

```
3️⃣ 자동 분석 시연 (30초)
   - 실시간 프로그레스 표시
```

## 💡 UX 안정화 원칙

### 1. 투명성 (Transparency)
- 모든 데이터 출처 공개
- 신뢰도 등급 표시
- 계산 과정 설명

### 2. 복원 가능성 (Recoverability)
- API 실패 시 대안 제공
- 데이터 부족 시 선택지 제공
- 언제든 이전 단계로 돌아갈 수 있음

### 3. 예측 가능성 (Predictability)
- 진행 상황 실시간 표시
- 예상 소요 시간 안내
- 다음 단계 명확히 제시

### 4. 용서 가능성 (Forgivingness)
- 에러를 에러로 보지 않음
- "이렇게 하면 해결됩니다" 제시
- 수기 입력 등 우회 경로 제공
