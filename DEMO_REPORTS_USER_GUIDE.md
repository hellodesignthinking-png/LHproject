## 🖨️ **PDF 출력 가이드**

### 방법 1: 브라우저 직접 인쇄 (권장)

#### Chrome/Edge
1. Demo Report URL 접속
2. **Ctrl + P** (Windows) 또는 **Cmd + P** (Mac)
3. 인쇄 설정:
   - **대상**: "PDF로 저장"
   - **레이아웃**: 세로
   - **용지 크기**: A4
   - **여백**: 기본값
   - **배경 그래픽**: ✅ 체크
   - **배율**: 100%
4. **저장** 클릭

#### Firefox
1. Demo Report URL 접속
2. **Ctrl + P** (Windows) 또는 **Cmd + P** (Mac)
3. 인쇄 설정:
   - **프린터**: "Microsoft Print to PDF" 또는 "Save as PDF"
   - **용지 방향**: 세로
   - **배경 인쇄**: ✅ 체크
4. **인쇄** 클릭

### 방법 2: 온라인 HTML to PDF 변환

1. **HTML 파일 저장**:
   - Demo Report 페이지에서 우클릭
   - "다른 이름으로 저장" → "웹페이지, 완료"
   
2. **PDF 변환 사이트 이용**:
   - https://html2pdf.com
   - https://pdfcrowd.com
   - 또는 기타 HTML to PDF 변환 서비스

### 방법 3: Python 스크립트 (개발자용)

```python
# weasyprint 사용
from weasyprint import HTML

# Demo Report URL to PDF
HTML('https://8090-ismcj42l609zyihh62150-ad490db5.sandbox.novita.ai/demo_gangnam_youth.html').write_pdf('gangnam_youth_report.pdf')
```

## 🐛 **문제 해결 (Troubleshooting)**

### 문제 1: 페이지가 로드되지 않음

**원인**: Demo Server가 중지됨

**해결**:
```bash
cd /home/user/webapp
cd generated_reports && python -m http.server 8090 &
```

### 문제 2: 일부 섹션이 비어 있음

**원인**: 데이터 생성 오류

**해결**:
```bash
cd /home/user/webapp
python generate_phase_11_14_demo_report.py
```

### 문제 3: PDF 출력 시 깨짐

**원인**: 브라우저 설정 문제

**해결**:
- 배경 그래픽 인쇄 옵션 ✅ 체크
- 배율 100% 설정
- A4 용지 크기 선택

### 문제 4: 모바일에서 레이아웃 깨짐

**원인**: 반응형 CSS 미적용

**해결**:
- 최신 버전 브라우저 사용
- 화면 회전 (가로 모드 권장)
